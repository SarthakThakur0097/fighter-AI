# UFC Analytics & Machine Learning Project - Context Document

## Project Overview
Building an end-to-end UFC analytics and machine-learning system to predict fight outcomes and identify betting/analytical edges. The system starts with custom web scraping, moves through data normalization and storage, and culminates in ML models for fight prediction.

## Current Status: Data Ingestion Phase

### Test Fighter
- **Currently testing on**: Jon Jones (28-1-0, 1 NC)
- **Purpose**: Validate complete scraping pipeline and database schema before scaling to full roster
- **Validation approach**: Manual verification of scraped data accuracy against source

## Technical Stack

### Data Sources
1. **Primary Source**: UFCStats.com (ufcstats.com)
   - Fighter profiles with career statistics
   - Detailed fight-by-fight breakdowns
   - Per-round striking statistics (unique to this source)
   
2. **Existing Database**: Already contains:
   - Fighter biographical data
   - Fight dates (`event_date`) 
   - Aggregated fight statistics
   - May need to merge/enhance with new scraper data

### Scraping Approach
- **Tool**: Node.js with node-fetch (Node 18+)
- **Why not Puppeteer**: UFCStats serves full HTML without JavaScript rendering; node-fetch is 10-50x faster
- **Strategy**: Scrape fighter profile → scrape each fight's detailed page

## Data Being Captured

### Fighter Profile Data
- Basic info: name, record, height, weight, reach, stance, DOB
- Career averages: SLpM, StrAcc, SApM, StrDef, TDAvg, TDAcc, TDDef, SubAvg
- Complete fight history with URLs

### Per-Fight Data
**High-level:**
- Result, opponent, event name, date, method, round, time
- Weight class, title bout status (regular/interim)
- Referee, judges, scorecards
- Fight bonuses (Performance, Fight, KO, Submission of the Night)

**Fight Totals Tables:**
- Knockdowns (KD)
- Significant strikes (landed/attempted/percentage)
- Total strikes (landed/attempted)
- Takedowns (landed/attempted/percentage)
- Submission attempts
- Reversals
- Control time

**Significant Strikes Breakdown:**
- By target: head, body, leg (landed/attempted)
- By position: distance, clinch, ground (landed/attempted)
- Body part percentages

**Per-Round Granularity** (unique capability):
- All striking stats broken down by round
- Enables analysis of:
  - Fighter momentum/adaptation
  - Cardio/performance degradation
  - Strategic adjustments between rounds

## Key Technical Decisions

### Why Node.js + node-fetch?
- UFCStats doesn't require JavaScript execution
- 10-50x faster than Puppeteer
- Lower memory footprint
- Easier deployment

### Why Not Scraping All Fighters Yet?
- Validate data quality first
- Ensure schema handles edge cases
- Avoid wasting time on bad data structure
- Can parallelize once proven

### Data Quality Philosophy
- Manual verification on test fighter
- Automated validation rules after proven
- Better to be slow and accurate than fast and wrong
